# 📱 حالة مشروع تطبيق المحادثة (WhatsApp Clone)

هذا الملف يحتوي على تتبع دقيق لجميع المراحل والمهام في مشروع تطبيق المحادثة باستخدام React و Firebase لكي نبقى على دراية بما تم إنجازه وما يجب القيام به خطوة بخطوة.

## ✅ ما تم إنجازه حتى الآن

- [x] **تأسيس المشروع:** إنشاء المشروع باستخدام Vite (React + JavaScript).
- [x] **المكتبات الأساسية:** تثبيت المكتبات (Firebase, Zustand, React Router DOM, Tailwind CSS, Lucide React).
- [x] **إعداد التصميم:** إعداد هيكل Tailwind CSS بالكامل (الملفات: `tailwind.config.js`, `postcss.config.js`, `src/index.css`).
- [x] **إعداد قاعدة البيانات:** إنشاء ملف تهيئة Firebase الأساسي (`src/firebase/config.js`) مع تجهيز كل من (Firestore, Storage, Auth, Realtime DB).
- [x] **إدارة حالة المكالمات:** إنشاء متجر `useCallStore.js` باستخدام Zustand لإدارة حالة المكالمات برمجياً من أي مكان في التطبيق.
- [x] **واجهة المستخدم المبدئية:** تصميم الهيكل الأولي للواجهة (UI Skeleton) لتطبيق المحادثة في `src/App.jsx`.

## ⏳ المهام المتبقية (خريطة الطريق)

### المرحلة الأولى: الأساسيات والمصادقة (Authentication)
- [x] إنشاء `AuthContext` لإدارة حالة تسجيل دخول المستخدم عبر التطبيق بشكل مركزي.
- [x] تصميم وبرمجة صفحة تسجيل الدخول وإنشاء الحساب (`Login.jsx` / `Register.jsx`).
- [x] ربط تسجيل الدخول مع `Firebase Authentication` (البريد/كلمة المرور، وحساب Google).
- [x] توجيه المستخدم (Routing): حماية صفحة المحادثات (Private Route) بحيث لا يدخلها سوى المستخدمين المسجلين.

### المرحلة الثانية: نظام المراسلة الأساسي (Core Chat)
- [x] تصميم مكون للبحث عن المستخدمين الآخرين لبدء محادثات فردية جديدة.
- [x] هندسة قاعدة بيانات Firestore (مجموعة `chats` ومجموعة `messages`).
- [x] تطبيق تقسيم البيانات (Pagination): جلب أول 20 رسالة فقط عند الدخول للمحادثة لحماية الحصة المجانية.
- [x] جلب الرسائل القديمة عند التمرير للأعلى (Scroll Up) باستخدام `startAfter`.
- [x] الاستماع للرسائل الجديدة وإرسالها لحظياً باستخدام `onSnapshot`.

### المرحلة الثالثة: إرسال الملفات والوسائط (Media & Location)
- [x] تصميم وتفعيل قائمة المرفقات في واجهة الدردشة (صور، مستندات، موقع، صوت).
- [x] دمج مكتبة `browser-image-compression` لضغط الصور إجبارياً قبل الرفع.
- [x] رفع الملفات المرفقة إلى `Firebase Storage` وحفظ الرابط في الـ Firestore.
- [x] تسجيل الملاحظات الصوتية (Voice Notes) عبر المتصفح باستخدام `MediaRecorder API` ورفعها.
- [x] التقاط الموقع الجغرافي (`Geolocation API`) للمستخدم وإرساله في الدردشة كرابط خرائط.

### المرحلة الرابعة: المكالمات الصوتية والمرئية (WebRTC Calls)
- [x] تصميم واجهة الاتصال (Call Modal) لعرضها عند وجود اتصال وارد أو صادر بالاعتماد على `useCallStore`.
- [x] استخدام مكتبة `simple-peer` لتبسيط ربط الـ P2P بين المتصلين.
- [x] كتابة آلية تبادل إشارات الاتصال (Signaling: Offer/Answer/ICE) باستخدام Socket.io.
- [x] دمج خوادم STUN للتعرف على الـ IP، وخادم TURN المجاني (مثل `openrelay.metered.ca`) كنسخة احتياطية للاتصال.
- [x] تفعيل أزرار الرد والإلغاء وتوصيل تدفق الفيديو والصوت (Media Streams).

### المرحلة الخامسة: تحديث الحالة والإشعارات (Presence & Notifications)
- [x] إدارة ظهور حالة الاتصال (Online / Offline / Last Seen) باستخدام ميزة `onDisconnect` المتوفرة في Realtime Database حصراً.
- [x] إعداد `Web Push API` لإرسال الإشعارات عبر المتصفح بطريقة أصلية Native لدعم مختلف المنصات (بما فيها iOS بقدر الإمكان).
- [x] إعداد دوال خلفية مجانية عبر Vercel (بدلاً من Cloud Functions) لإرسال إشعارات الدفع بشكل آمن.
